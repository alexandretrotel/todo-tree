# Architecture

This document describes the high-level architecture and workflow of the `todo-tree` CLI tool.

## Overview

`todo-tree` is a command-line tool that scans source code for TODO-style comments (e.g., `TODO`, `FIXME`, `BUG`) and displays them in a structured format. It's inspired by the VS Code "Todo Tree" extension.

## Commands

| Command | Description |
|---------|-------------|
| `scan`  | Scan directories for TODO comments (default) |
| `list`  | List TODOs in flat format |
| `tags`  | Show or manage configured tags |
| `init`  | Create a new `.todorc` config file |
| `stats` | Show statistics about TODOs |

## Priority Levels

Tags are mapped to priority levels for sorting and colorization:

| Priority | Tags | Color |
|----------|------|-------|
| Critical | `BUG`, `FIXME`, `XXX` | Red |
| High | `HACK`, `WARN`, `WARNING` | Yellow |
| Medium | `TODO`, `PERF` | Cyan |
| Low | `NOTE`, `INFO`, `IDEA` | Green |

## Configuration Resolution

Configuration is loaded with the following precedence (highest first):

1. CLI arguments
2. `--config` specified file
3. `.todorc` / `.todorc.json` / `.todorc.yaml` in current directory
4. Config files in parent directories (walking up)
5. `~/.config/todo-tree/config.{json,yaml}` (global)
6. Built-in defaults
